# 八字排盘、大运流年与神煞计算说明

> 本文是 `paipan` 项目的辅助算法文档，主要说明当前代码采用的计算口径，并补充流月、流日的常见推算方法。
>
> 命理流派对换年、换日、早晚子时、真太阳时、起运折算和神煞表均可能存在差异。本文优先以本仓库 `lib/class.paipan.php` 与 `js/paipan.js` 的实现为准，不把命理结论视为科学事实，也不建议仅凭神煞判断吉凶。

## 1. 基础编码

项目内部使用从 0 开始的数字表示天干、地支和六十甲子：

```text
天干：0甲 1乙 2丙 3丁 4戊 5己 6庚 7辛 8壬 9癸
地支：0子 1丑 2寅 3卯 4辰 5巳 6午 7未 8申 9酉 10戌 11亥
```

六十甲子的序号为 `0..59`，其中 `0=甲子`、`1=乙丑`，依次循环到 `59=癸亥`。

一组天干、地支能组成合法干支的前提是阴阳一致，即：

```text
tg % 2 == dz % 2
```

本项目通过 `GZ(tg, dz)` 把一柱的天干、地支转换为六十甲子序号：

```text
gz = ((10 + tg - dz) % 10) / 2 * 12 + dz
```

PHP 调用：

```php
$gzIndex = $p->GZ($tg, $dz);
```

## 2. 八字排盘总流程

八字排盘的入口为：

```php
$fm = $p->fatemaps($xb, $yy, $mm, $dd, $hh, $mt, $ss, $J, $W);
```

参数含义：

- `$xb`：性别，`0=男`，`1=女`；
- `$yy/$mm/$dd`：公历年月日；
- `$hh/$mt/$ss`：时、分、秒；
- `$J`：出生地经度，东经为正、西经为负；
- `$W`：出生地纬度，北纬为正、南纬为负；
- 不传 `$J` 时，直接按输入的标准时间排盘；
- 传入 `$J` 时，先换算地方平太阳时和真太阳时，再计算四柱。

主要步骤如下：

1. 校验公历日期；
2. 把公历时刻转换为儒略日；
3. 按需要转换地方平太阳时、真太阳时；
4. 计算二十四节气时刻；
5. 以立春确定年柱；
6. 以十二个“节”确定月柱；
7. 由儒略日连续序数确定日柱；
8. 由日干和时辰确定时柱；
9. 计算阴阳、五行、藏干、十神；
10. 根据性别与年干阴阳确定大运顺逆；
11. 计算起运时间、大运和流年；
12. 计算神煞及刑冲合害关系。

核心返回值包括：

```text
sz                    四柱文字，例如 [庚午, 戊子, 丙寅, 甲午]
tg / dz               四柱天干、地支数字数组
nl                     农历生日
nyy / nwx              阴阳、五行数量
bctg / bzcg            藏干及藏干十神
qyy / qyy_desc         起运年份、起运描述
dy                     大运及各步大运下的流年
shensha                按柱输出的神煞文本
shensha_detail         神煞命中明细
shensha_by_pillar      按年、月、日、时柱聚合的神煞
```

## 3. 年柱计算

### 3.1 年柱以立春为界

干支纪年与农历正月初一纪年不是同一口径。本项目以**立春时刻**作为八字年柱的分界：

- 出生时刻在当年立春之前：仍属于上一干支年；
- 出生时刻达到或超过当年立春：进入新的干支年。

因此，公历 1 月 1 日并不换年，农历正月初一也不一定换八字年柱。

### 3.2 年柱序号

代码先确定节气年 `ty`，再计算：

```text
ygz = mod(ty + 4712 + 24, 60)
年干 = ygz % 10
年支 = ygz % 12
```

这里的 `mod` 应使用数学意义上的非负取模，以兼容公元前年份。

## 4. 月柱计算

### 4.1 月柱按节气月，不按农历月

八字月柱采用干支历的节气月。每月以“节”交接，不以农历初一交接：

| 月支 | 起始节 | 大致公历范围 |
| --- | --- | --- |
| 寅 | 立春 | 2 月上旬至 3 月上旬 |
| 卯 | 惊蛰 | 3 月上旬至 4 月上旬 |
| 辰 | 清明 | 4 月上旬至 5 月上旬 |
| 巳 | 立夏 | 5 月上旬至 6 月上旬 |
| 午 | 芒种 | 6 月上旬至 7 月上旬 |
| 未 | 小暑 | 7 月上旬至 8 月上旬 |
| 申 | 立秋 | 8 月上旬至 9 月上旬 |
| 酉 | 白露 | 9 月上旬至 10 月上旬 |
| 戌 | 寒露 | 10 月上旬至 11 月上旬 |
| 亥 | 立冬 | 11 月上旬至 12 月上旬 |
| 子 | 大雪 | 12 月上旬至次年 1 月上旬 |
| 丑 | 小寒 | 1 月上旬至立春 |

交节日必须比较具体到时、分、秒；不能仅按公历日期或固定月份判断。

### 4.2 月干——五虎遁

月支固定从寅月开始，月干由年干决定：

```text
甲己之年丙作首
乙庚之岁戊为头
丙辛岁首寻庚起
丁壬壬位顺行流
若言戊癸何方发
甲寅之上好追求
```

即：

| 年干 | 寅月月干 |
| --- | --- |
| 甲、己 | 丙 |
| 乙、庚 | 戊 |
| 丙、辛 | 庚 |
| 丁、壬 | 壬 |
| 戊、癸 | 甲 |

从寅月起，天干、地支分别顺推一位，直到丑月。

项目中可使用：

```php
$months = $p->MGZ($yearGzIndex);
```

获取某年柱下全部合法月柱。

## 5. 日柱计算

### 5.1 由儒略日连续推算

项目先将出生时刻转换为儒略日 `spcjd`，再把儒略日起点从正午调整到民用日界，计算六十日循环：

```text
jda   = spcjd + 0.5
thes  = 小数日对应秒数 + 3600 秒
dayjd = floor(jda) + thes / 86400
dgz   = mod(floor(dayjd + 49), 60)
```

随后：

```text
日干 = dgz % 10
日支 = dgz % 12
```

`+49` 是本项目采用的儒略日与六十甲子基准偏移。

### 5.2 换日与早晚子时

当前 PHP 类中：

```php
$p->zwz = true;
```

表示区分早子时与晚子时。开启后，`23:00—24:00` 的日柱按项目当前规则前移一柱；关闭后则不做这一调整。

不同流派常见口径包括：

1. 子初换日：23:00 换日；
2. 子正换日：00:00 换日；
3. 早晚子时分别处理。

跨 23:00、00:00 的案例，应在项目中固定一种口径并写入测试，避免排盘结果在升级后变化。

## 6. 时柱计算

### 6.1 时支划分

通常每两个小时为一个时辰：

| 时支 | 时间 |
| --- | --- |
| 子 | 23:00—00:59 |
| 丑 | 01:00—02:59 |
| 寅 | 03:00—04:59 |
| 卯 | 05:00—06:59 |
| 辰 | 07:00—08:59 |
| 巳 | 09:00—10:59 |
| 午 | 11:00—12:59 |
| 未 | 13:00—14:59 |
| 申 | 15:00—16:59 |
| 酉 | 17:00—18:59 |
| 戌 | 19:00—20:59 |
| 亥 | 21:00—22:59 |

若采用真太阳时，应先完成时间换算，再判断时支。

### 6.2 时干——五鼠遁

时干由日干决定：

```text
甲己还加甲
乙庚丙作初
丙辛从戊起
丁壬庚子居
戊癸何方发
壬子是真途
```

| 日干 | 子时时干 |
| --- | --- |
| 甲、己 | 甲 |
| 乙、庚 | 丙 |
| 丙、辛 | 戊 |
| 丁、壬 | 庚 |
| 戊、癸 | 壬 |

项目中可使用：

```php
$hours = $p->HGZ($dayGzIndex);
```

获取某日柱下全部合法时柱。

## 7. 真太阳时

标准时间是时区统一时间，不等于出生地的地方太阳时。本项目在传入经纬度时执行两层修正：

1. 经度修正为地方平太阳时；
2. 根据太阳升落的天文计算修正为真太阳时。

经度的粗略修正关系为：

```text
地方平太阳时修正分钟数 ≈ (出生地经度 - 120°) × 4
```

例如东经 110° 相对东经 120° 的北京时间理论上约慢 40 分钟。

本项目当前方案是：真太阳时转换后，年月日时四柱都按转换后的时刻计算，但节气数据本身仍使用北京时间口径。临近立春、交节、23:00 或时辰边界时，应特别标注采用的方案。

## 8. 大运计算

## 8.1 顺排与逆排

项目按照性别与年干阴阳确定大运方向：

- 阳年男、阴年女：顺排；
- 阴年男、阳年女：逆排。

代码中天干编号偶数为阳、奇数为阴，因此可以写为：

```text
顺排 = 男且年干为阳，或女且年干为阴
逆排 = 男且年干为阴，或女且年干为阳
```

注意：部分流派会以日干阴阳、年柱阴阳或其他规则确定顺逆，不能混用。

## 8.2 起运距离

先找到出生时刻所在节气月的前后两个“节”：

```text
xf = 出生时刻 - 前一个节

yf = 后一个节 - 出生时刻
```

- 顺排取出生后到下一个节的距离 `yf`；
- 逆排取前一个节到出生时刻的距离 `xf`。

### 粗略法

默认：

```php
$p->pdy = false;
```

按一年 360 天、每个节气月按 30 天折算：

```text
起运折算天数 zf = 360 × 10 × 节气距离 / 30
```

它等价于传统常见的“三天折一年”：

```text
3 天 = 1 年
1 天 = 4 个月
1 个时辰约 = 10 天
1 小时约 = 5 天
```

### 精确法

设置：

```php
$p->pdy = true;
```

使用回归年长度 `365.24244475`，并按出生点在完整节气月中的比例折算十年：

```text
顺排：zf = 回归年 × 10 × yf / (xf + yf)
逆排：zf = 回归年 × 10 × xf / (xf + yf)
```

最后把 `zf` 拆成年、月、日，形成：

```text
出生后 X 年 X 个月 X 天起运
```

## 8.3 大运干支

先把出生月柱转换成六十甲子序号 `mgz`：

```text
mgz = ((10 + 月干 - 月支) % 10) / 2 × 12 + 月支
```

每步大运十年：

- 顺排：从出生月柱的下一柱开始；
- 逆排：从出生月柱的上一柱开始。

第 `k` 步大运（`k` 从 0 开始）可表示为：

```text
顺排大运序号 = mod(mgz + k + 1, 60)
逆排大运序号 = mod(mgz - k - 1, 60)
```

项目默认生成 9 步大运，并在 `dy[k]` 中返回：

```text
zqage / zboz   此步大运起止虚岁
zfma / zfmb    大运天干、地支文字
zfman / zfmbn  大运天干、地支编号
nzsc           十二长生文字
ly              此步大运包含的十个流年
```

## 9. 流年计算

流年就是某一节气年的干支。八字口径通常仍以立春换年，而不是公历 1 月 1 日或农历正月初一。

项目在出生年柱基础上连续推算 90 个流年。第 `j` 个流年：

```text
流年干 = (出生年干 + j + 起运虚岁偏移) % 10
流年支 = (出生年支 + j + 起运虚岁偏移) % 12
```

返回字段：

```text
age    流年虚岁
year   流年对应年份
lyean  流年天干编号
lyebn  流年地支编号
lye    流年干支文字
```

如果只知道公历年份 `Y`，也可按年柱相同的六十甲子公式计算流年干支；但在立春前后的日期，必须先判断该时刻属于前一节气年还是新节气年。

## 10. 流月计算

当前 `fatemaps()` 主要返回本命、大运和流年，未直接展开流月数组。流月可以沿用本命月柱的规则计算。

### 10.1 流月边界

流月同样按十二个“节”切换：

```text
立春寅月、惊蛰卯月、清明辰月……小寒丑月
```

不要直接使用农历月份，也不要简单按公历每月 1 日切换。

### 10.2 流月月干

先取得流年天干，再用五虎遁确定该流年寅月的月干，之后逐月顺推。

若天干编号为 `yearGan`，寅月天干可写为：

```text
寅月干 = ((yearGan % 5) × 2 + 2) % 10
```

其中：

```text
甲/己年 -> 丙寅
乙/庚年 -> 戊寅
丙/辛年 -> 庚寅
丁/壬年 -> 壬寅
戊/癸年 -> 甲寅
```

设 `monthOffset=0` 表示寅月，`1` 表示卯月，依次到 `11` 表示丑月：

```text
流月干 = (寅月干 + monthOffset) % 10
流月支 = (2 + monthOffset) % 12
```

实现时应先通过节气时刻确认 `monthOffset`，再组合流月干支。

## 11. 流日计算

流日与本命日柱使用同一套儒略日算法。给定目标公历时刻：

1. 根据采用的标准时间或真太阳时口径修正目标时刻；
2. 转换为儒略日；
3. 使用 `GetGZ()` 计算目标时刻的日柱；
4. 取返回数组中的 `tg[2]`、`dz[2]`。

PHP 示例：

```php
[$tg, $dz, $ob] = $p->GetGZ(2026, 7, 10, 12, 0, 0);
$flowDay = $p->ctg[$tg[2]] . $p->cdz[$dz[2]];
```

如果还需要流时，则以该流日的日干套用五鼠遁，根据目标时辰求出时柱即可。

> 注意：如果目标时刻在 23:00—00:59，流日、流时必须和 `$p->zwz` 的早晚子时配置保持一致。

## 12. 神煞计算总则

项目通过：

```php
$ss = $p->GetShensha($tg, $dz);
```

计算本命四柱神煞，返回：

```text
items       每个神煞的全部命中明细
by_pillar   按命中柱位聚合
lines       兼容展示的四行文本
```

每条明细会记录：

```text
target_pillar       神煞落在哪一柱
trigger_pillar      由哪一柱触发
trigger_type        以天干、地支或日柱触发
trigger_value       触发值
rule                规则说明
```

当前实现按以下来源分类：

1. 日干起；
2. 年干起；
3. 月支起；
4. 年支起；
5. 年支或日支所属三合组起；
6. 日支起；
7. 地支自身判断；
8. 日柱特殊判断。

## 17. 三合组神煞

当前实现只允许**年支和日支**作为触发源，月支、时支不触发。先把触发支归入三合组：

```text
申子辰 -> 水局组
巳酉丑 -> 金局组
```

## 18. 日支及特殊日柱神煞

### 18.1 元辰

当前代码采用：

```text
目标地支 = (日支 + 7) % 12
```

并且只检查时支，不落年、月、日柱。

### 18.2 空亡

空亡按日柱所在旬确定：

| 日柱所在旬 | 空亡 |
| --- | --- |
| 甲子旬 | 戌、亥 |
| 甲戌旬 | 申、酉 |
| 甲申旬 | 午、未 |
| 甲午旬 | 辰、巳 |
| 甲辰旬 | 寅、卯 |
| 甲寅旬 | 子、丑 |

代码用：

```text
旬序号 = floor(日柱六十甲子序号 / 10)
```

取得对应的两个空亡地支，再检查四柱。

### 18.3 魁罡

当前实现认定以下四个日柱为魁罡：

```text
庚辰、壬辰、庚戌、壬戌
```

### 18.4 天罗、地网

当前实现按地���自身判断：

```text
四柱见戌或亥 -> 天罗
四柱见辰或巳 -> 地网
```

这是一种简化、明确的项目口径；其他资料可能附加男女、年命或纳音条件。

## 19. 把神煞扩展到大运、流年、流月、流日

`GetShensha()` 的输入是天干数组和地支数组。当前方法主要面向四柱本命：

```php
$ss = $p->GetShensha($fm['tg'], $fm['dz']);
```

若要分析大运、流年、流月、流日，建议区分两类需求。

### 19.1 重新组成扩展柱位

例如：

```text
0 年柱
1 月柱
2 日柱
3 时柱
4 大运
5 流年
6 流月
7 流日
8 流时
```

本项目的 `GetGX()` 已允许传入超过四柱的数组，用于分析本命、大运、流年等之间的刑冲合害。

但 `GetShensha()` 当前固定按四柱设计，直接传入九柱不会完整识别新增柱位。若要扩展，建议：

1. 把柱名、目标柱范围改为参数；
2. 明确哪些神煞只查本命、哪些允许查运年；
3. 保留触发柱与命中柱，避免“本命神煞”和“岁运神煞”混为一谈；
4. 为每个神煞建立表驱动规则和独立测试用例。

### 19.2 仅以岁运支检查目标

另一种较安全的实现是：保留本命的年干、年支、月支、日干、日支作为触发源，只把大运、流年、流月、流日作为新增目标柱检查。

例如“日干甲，流年地支为丑”，可记录为日干起天乙贵人落流年；但不应默认让流年支反向触发所有年支、日支类神煞，除非业务明确采用该口径。

## 20. 刑冲合害与岁运分析

神煞之外，项目还提供：

```php
$relations = $p->GetGX($tg, $dz);
```

可识别天干冲合，以及地支的冲、刑、自刑、六合、三合、半合、拱合、三会、拱会、暗合、相害等关系。

分析岁运时，可把数组扩展为：

```php
$tg = [年干, 月干, 日干, 时干, 大运干, 流年干, 流月干, 流日干, 流时干];
$dz = [年支, 月支, 日支, 时支, 大运支, 流年支, 流月支, 流日支, 流时支];
$relations = $p->GetGX($tg, $dz);
```

建议在展示层同时输出参与关系的柱位，例如：

```text
日柱地支 + 流年地支：寅申冲
月柱天干 + 大运天干：甲己合化土
```

注意“存在组合”不等于必然“合化成功”。是否得令、得地、透干、有无冲破等属于后续命理解释层，不应混入基础历法计算层。

## 21. 推荐的程序分层

为了避免不同流派规则互相污染，建议分为四层：

### 21.1 历法层

负责：

- 公历、农历、儒略日转换；
- 节气计算；
- 标准时间、地方平太阳时、真太阳时转换。

### 21.2 干支层

负责：

- 年、月、日、时四柱；
- 大运、流年、流月、流日、流时；
- 五虎遁、五鼠遁；
- 早晚子时配置。

### 21.3 规则层

负责：

- 十神、藏干、十二长生；
- 神煞；
- 刑冲合害；
- 各流派可配置规则。

### 21.4 解释层

负责：

- 命局、岁运组合的文字说明；
- 明确展示采用的流派、边界和配置；
- 不修改历法层已经确定的时间与干支结果。

## 22. 测试建议

至少覆盖以下边界：

1. 立春前 1 秒、立春时刻、立春后 1 秒；
2. 每个交节时刻前后；
3. 22:59:59、23:00:00、23:59:59、00:00:00；
4. 标准时间与真太阳时跨日；
5. 真太阳时跨立春或跨节；
6. 男女性别和十个年干的顺逆大运；
7. 粗略起运与精确起运；
8. 六十日循环的连续日期；
9. 闰年、公历改革日期及公元前年份；

新增或调整神煞规则时，应同步增加验证案例。
